# Luden - Structural Math Problem Matcher

A Flask web app that takes a math equation entered via an interactive keyboard, parses it into an Abstract Syntax Tree (AST), and finds the most structurally similar problems from a dataset scraped from Paul's Online Math Notes.

---

## What it does

You type a math equation using an on-screen keyboard, and the app finds algebra problems with the same *structure* — not just similar-looking ones. The key idea is structural invariance: `5x + 2 = 12` and `4y + 7 = 19` are the same *type* of problem. Numbers get replaced with `C` and variables with `X1, X2, ...`, so matching is based purely on mathematical shape.

## Tech Stack

| Component        | Library / Tool                          |
| ---------------- | --------------------------------------- |
| Language         | Python 3                                |
| Web Framework    | Flask                                   |
| Math Parsing     | latex2sympy2_extended + SymPy           |
| Similarity Score | Custom hierarchical scorer + difflib    |
| Math Input       | MathLive (browser virtual keyboard)     |
| Math Rendering   | MathJax 3                               |
| Data Scraping    | requests + BeautifulSoup4               |
| Data Source      | Paul's Online Math Notes (Algebra)      |

---

## How it works

```
User Input (MathLive keyboard)
    → LaTeX String
    → AST Normalization (latex2sympy2 + custom prefix serializer)
    → Hierarchical Similarity Scoring against AST Database
    → Top 10 Structural Matches (rendered with MathJax)
```

Matches are ranked by a weighted score:
- **45%** — top-level operator match (same problem type)
- **30%** — structural operator overlap (trig, log, sqrt, etc.)
- **25%** — token sequence similarity via `difflib`

Results are color-coded: green (≥ 0.70), yellow (0.45–0.69), red (< 0.45).

## Project Structure

```
Luden/
├── app.py               # Flask server — scoring logic and route handler
├── build_ast.py         # Parses LaTeX into prefix AST signatures → ast_database.json
├── scraper.py           # Scrapes algebra problems from Paul's Online Notes → algebra_data.json
├── algebra_data.json    # Raw scraped dataset (1,122 problems) — committed so you don't need to re-scrape
├── ast_database.json    # Generated by build_ast.py — not committed, must be built locally
├── templates/
│   └── index.html       # Frontend — MathLive input, MathJax rendering, dark mode
└── requirements.txt
```


---

## Running

### Run the app

```bash
python app.py
```

Go to `http://127.0.0.1:5000`, enter an equation with the math keyboard, and click **Find Matches**.

### Build the AST database (required on first run)

```bash
python build_ast.py
```

Parses `algebra_data.json` into normalized prefix AST signatures and writes `ast_database.json`. Re-run if you change the parser logic or re-scrape.

### (Optional) Re-scrape the dataset

```bash
python scraper.py
```

Crawls Paul's Online Math Notes and overwrites `algebra_data.json`. Only needed if you want fresh data.

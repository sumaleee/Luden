# Luden - Structural Math Problem Matcher

A Flask web app that takes a math equation entered via an interactive keyboard or uploaded image, parses it into an Abstract Syntax Tree (AST), and finds the most structurally similar problems from a dataset scraped from Paul's Online Math Notes.

---

## What it does

You type a math equation using an on-screen keyboard (or photograph one and use the built-in OCR), and the app finds algebra problems with the same *structure* — not just similar-looking ones. The key idea is structural invariance: `5x + 2 = 12` and `4y + 7 = 19` are the same *type* of problem. Numbers get replaced with `C` and variables with `X1, X2, ...`, so matching is based purely on mathematical shape.

## How it works

```
User Input
    ├── MathLive keyboard → LaTeX string
    └── Image upload → drag-to-crop → pix2tex OCR → LaTeX string
            ↓
    AST Normalization (latex2sympy2 + custom prefix serializer)
            ↓
    Hierarchical Similarity Scoring against AST Database
            ↓
    Top 10 Structural Matches (rendered with MathJax)
```

Matches are ranked by a weighted score:
- **45%** — top-level operator match (same problem type)
- **30%** — structural operator overlap (trig, log, sqrt, etc.)
- **25%** — token sequence similarity via `difflib`

Results are color-coded: green (≥ 0.70), yellow (0.45–0.69), red (< 0.45).

---

## OCR / Image Input

Drop or click-to-upload an image onto the drop zone above the math field. A crop UI appears — select the region containing the equation, then click **Crop & Scan**. The cropped region is sent to the local pix2tex model, which returns a LaTeX string that is injected directly into the math field.

The OCR model loads on first use (~5s), then stays in memory for the session.

---

## Scraper behaviour

Each problem `<li>` on Paul's site can contain multiple LaTeX expressions (e.g. "Add `10x^5+2x^3-1` to `8x^4-x^3+16x^2`" has two separate polynomials). The scraper extracts **every** LaTeX expression from each problem element and saves each as its own dataset entry, so all expressions get individual AST signatures and are independently matchable.

## Project Structure

```
Luden/
├── app.py               # Flask server — scoring logic, /ocr route, main route handler
├── build_ast.py         # Parses LaTeX into prefix AST signatures → ast_database.json
├── scraper.py           # Scrapes algebra problems from Paul's Online Notes → algebra_data.json
├── algebra_data.json    # Raw scraped dataset — committed so you don't need to re-scrape
├── ast_database.json    # Generated by build_ast.py — not committed, must be built locally
├── templates/
│   └── index.html       # Frontend — MathLive input, OCR drop zone, MathJax rendering, dark mode
└── requirements.txt
```

## Running

### Build the AST database (required on first run)

```bash
python build_ast.py
```

Parses `algebra_data.json` into normalized prefix AST signatures and writes `ast_database.json`. Re-run if you change the parser logic or re-scrape.

### Run the app

```bash
python app.py
```

Go to `http://127.0.0.1:5000`, enter an equation with the math keyboard or upload an image, and click **Find Matches**.

### (Optional) Re-scrape the dataset

```bash
python scraper.py
```

Crawls Paul's Online Math Notes and overwrites `algebra_data.json`. Only needed if you want fresh data. Re-run `build_ast.py` afterwards.
